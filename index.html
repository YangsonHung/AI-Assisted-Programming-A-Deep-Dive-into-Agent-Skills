01

AI 编程认知基础

从传统编程到 AI 辅助开发的范式转变

传统编程开发的痛点

作为开发者，经常会遇到这样的场景：

上下文切换成本高

需求理解 → 技术选型 → 代码实现 → 质量验证的切换过程中，每次都要重新构建认知框架

知识传递效率低

项目规范、架构经验分散在文档和个人经验中，新成员上手或跨模块开发时处处碰壁

开发流程割裂

需求 → 设计 → 编码 → 审查各环节串行传递，信息易失真且反馈滞后

核心问题：开发者需要同时扮演"架构师"和"码农"的角色，既要设计方案，又要手动实现

传统编程 vs AI 编程

两者代表了两种不同的软件开发范式，并非替代关系，而是互补演进

维度 传统编程 AI 编程
交互方式 代码优先，通过编程语言与计算机对话 自然语言优先，通过描述需求与 AI 协作
代码来源 开发者逐行编写，依赖记忆和经验 AI 根据需求生成，开发者审核优化
知识依赖 依赖开发者掌握的语言框架和 API AI 具备广泛知识，开发者更多扮演指导角色
角色定位 开发者是执行者，全程参与 开发者是指挥者，关注决策和验收
迭代速度 重复性代码编写耗时较长 代码生成快，聚焦于创造性工作

开发流程对比

传统编程流程

需求分析
技术设计
手动编码（核心环节）
单元测试 → 代码审查 → 部署

AI 编程流程

需求分析
自然语言描述
AI 代码生成
开发者审查优化

关键变化：AI 编程引入迭代反馈机制："生成 → 审查 → 反馈 → 优化"的敏捷循环

AI 编程的八个阶段

Steve Yegge（前谷歌/亚马逊工程师）在 2026 年提出的 AI 编程成熟度模型

1

古典编程

无 AI 辅助，纯手写代码

2

编辑器 AI 插件

Copilot、Cursor 等提供代码补全和问题解答

3

YOLO 模式

信任 AI，放任其产出，一路单击 Yes

4

AI 主导开发

AI 工作占比大，手动编码仅用于比对代码

AI 编程的八个阶段（续）

5

命令行模式

脱离编辑器，直接使用命令行，只关心完成度

6

多 AI 并行

开 3-5 个 AI 窗口并行编程

7

AI 集群

AI 窗口超过 10 个，达到人工管理极限

8

AI 任务管理器

指挥成百上千个 AI 同时工作（Steve Yegge 所处阶段，开发了 Gas Town）

为什么需要 AI 编程辅助？

个人层面

释放创造力
降低记忆负担

团队层面

缩小技能差距
加速知识传递

业务层面

加速产品迭代
降低试错成本

行业层面

缓解人才短缺
推动生产力跃迁

AI 编程辅助开发不是「选项」，而是应对多重挑战的「必选项」

Plan 模式：拆解复杂问题

借鉴项目管理中的 WBS（工作分解结构）思想，将大问题拆分成小问题

1. 需求分析与模块划分

把整体需求分解为独立的功能模块

2. 技术方案设计

为每个模块确定实现思路和技术选型

3. 任务优先级排序

根据依赖关系和重要性确定实现顺序

提示词明确四件事：功能边界和目标、技术约束和规范、输出格式和交付物、分阶段的实现计划

AI 编程的局限性与挑战

需求表达不完整

开发者说"实现一个商家信息查询接口"，AI 生成了基础的 CRUD 代码，但没有考虑数据权限、数据脱敏、缓存策略等实际业务需求

上下文管理混乱

对话持续多轮后，AI 开始忘记前面确定的设计决策，如"使用 MyBatis-Plus"被擅自改成 JPA Repository

迭代反馈滞后

等 AI 生成完整的代码后才发现方向不对，如数据库表设计与现有架构冲突，不得不从头再来，浪费大量时间

挑战 应对方法
需求表达不完整 使用 Plan 模式分阶段确认，使用 Agent Skills 沉淀需求模板
上下文管理混乱 控制单次对话粒度，定期回顾设计决策，使用 CLAUDE.md 传递上下文
迭代反馈滞后 先让 AI 生成接口定义和框架，确认后再实现细节

Agent Skills 核心概念

理解 Skills 的本质与技术生态

什么是 Agent Skills？

Skills 是模块化的、自包含的能力包，通过提供专门的知识、工作流和工具来扩展 AI Agent 的能力。它们就像是针对特定领域或任务的"入职指南"，将通用的 AI Agent 转变为具备专业程序性知识的专家。

Skills 能提供什么

  • 专业工作流：自动多步骤程序
  • 工具集成：处理特定文件格式或 API
  • 领域专业知识：公司特定的架构和业务逻辑
  • 打包资源：脚本、参考资料和资产

通过 Skills 能做什么

  • 封装专业知识为可复用的技能
  • 提供标准化的任务处理流程
  • 扩展 AI 处理更专业复杂任务的能力
  • 在团队成员间共享专业知识

Prompt vs Skills vs MCP

Prompt

Prompt Engineering

「对话」

关注点：如何描述需求

抽象层级：顶层交互

持久化：会话级（临时）

Agent Skills

能力包

「技能手册」

关注点：能做什么

抽象层级：中层能力

持久化：文件级（持久）

MCP

Model Context Protocol

「USB 接口」

关注点：如何连接

抽象层级：底层协议

持久化：服务级（持久）

调用链路：用户 Prompt → Skills（解析意图）→ MCP（调用工具）→ 执行结果 → 返回用户

Skills 标准规范

每个 Skill 位于 .claude/skills 目录下，由 SKILL.md 和可选资源组成

skill-name/
SKILL.md # 必需：技能说明文档
scripts/ # 可选：可执行脚本
example.py
helper.sh
references/ # 可选：参考文档
schema.md
api_docs.md
assets/ # 可选：输出资源（模板、图片等）

scripts/

需要确定性可靠性的可执行代码

references/

需要时加载到上下文的文档材料

assets/

在输出中使用的文件（无需加载）

Agent Skills 实战应用

从理论到实践的完整指南

创建 Agent Skills

创建 Skill 有两种方式：手写和使用工具

手写 Skills

适合学习原理和创建简单技能

my-skill/
└── SKILL.md # 必需文件

使用 skill-creator

快速生成标准化结构，适合批量创建

# 通过 Claude Code 安装
/skill skill-creator

# 启动工具
skill-creator create

前端框架 Skills 最佳实践

React 和 Vue 官方或社区已发布专属 Skills，标志着 AI 编程进入新阶段

React Skills

  • 组件设计：函数组件 + Hooks 规范
  • 状态管理：useState、Zustand、Redux Toolkit
  • 性能优化：useMemo、useCallback、代码分割
  • 样式方案：CSS Modules、Tailwind CSS

Vue Skills

  • 组合式 API：script setup、ref、reactive
  • 响应式系统：watch、watchEffect、computed
  • 组件通信：props、emits、provide/inject、Pinia
  • 路由规范：Vue Router 配置、导航守卫

关键要点：前端官方参与 Skill 建设，意味着 AI 助手正从"代码生成器"进化为"理解框架规范、工程经验的虚拟同事"

精选 Skills 推荐

skill-creator

技能生成器 - Agent 的元技能

通过引导式对话，将想法或 SOP 自动转化为符合标准的 SKILL.md 文件

frontend-design

前端设计优化

解决 AI 生成页面"千篇一律"的问题，提供独特字体组合和配色方案

superpowers

开发工作流框架

规范驱动开发（SDD），包含 brainstorm、write-plan、execute-plan 三板斧

总结与展望

AI 编程时代的机遇与挑战

关键要点回顾

第一章

AI 编程不是替代开发者，而是放大开发者能力，从"执行者"向"指挥者"转变

第二章

Agent Skills 是封装专业知识、工作流和工具的能力单元，可与 MCP 协同

第三章

Skills 可手写或用 skill-creator 创建，前端官方已积极参与 AI 编程规范建设

能力跃迁路径：传统编程 → AI 辅助 → AI 协作 → Agent 驱动

团队应用建议

起步阶段

快速上手

  • 安装 AI 编程工具：Cursor、Claude Code、VSCode + Copilot
  • 编写 CLAUDE.md 传递项目上下文和规范
  • 试用官方 Skills：frontend-design、skill-creator
建设阶段

规范沉淀

  • 沉淀团队规范：将编码规范、最佳实践固化为文档
  • 创建项目 Skills：用 skill-creator 生成项目专属 Skills
  • 建立反馈机制：收集 AI 编程中的问题和改进建议
规模化阶段

全面推广

  • 分享与复用：将优秀 Skills 在团队/社区分享
  • 持续优化：根据项目反馈迭代 Skills 内容
  • 培训推广：组织培训，让更多成员掌握 AI 编程

避坑指南

不要做

  • 一次性让 AI 生成完整模块（容易失控）
  • 忽视 AI 生成代码的审查
  • 过度依赖 AI 而丧失独立思考能力

应该做

  • 保持对代码的主导权
  • 使用 Plan 模式拆解复杂问题
  • 持续学习和优化 Prompt 技巧

未来发展趋势

专业化

官方 Skills 覆盖更多框架和场景

自动化

从人工编写 Skills 到 AI 自动生成 Skills

协作化

Skills 市场形成，开发者可买卖共享

标准化

行业统一的 Skills 规范和接口标准

过去

程序员 = 编码执行者

未来

程序员 = AI 指挥者 + 架构设计师 + 质量把关人

常见问题 Q&A

AI 编程会取代程序员吗？
不会。AI 是工具，不是替代者。AI 擅长执行重复任务，但需求理解、架构设计、代码审查仍需人类。未来的优秀程序员是那些能驾驭 AI 的人。
如何选择 AI 编程工具？
个人/小团队：VSCode + Copilot；中型项目：Cursor；大型项目/企业：Claude Code + Skills
Skills 和 MCP 都要学吗？
建议顺序：1. 先掌握 Prompt（入门简单）2. 再学习 Skills（提升复用性）3. 最后了解 MCP（深度集成）
